Web-Based Voting System for Debre Tabor University Student Union

 Project Description

The Web-Based Voting System for Debre Tabor University Student Union is an online election platform designed to improve the efficiency,
transparency, and reliability of student union elections.
The system allows students to securely participate in voting processes using a web interface while enabling election officers and
administrators to manage candidates, voters, and election results.

This system replaces the traditional manual voting process with a digital solution that minimizes human errors, reduces time consumption,
and enhances election transparency. It provides a centralized platform where authorized users can conduct, monitor, and manage elections in an organized manner.

The system is developed using PHP, MySQL, HTML, CSS, and JavaScript, and runs on a web server environment such as XAMPP. It supports different user roles
including administrators, Election Committee,Discipline Committee,Voter,Candidate, and students, each with specific permissions and responsibilities within the voting process.
The platform ensures that each registered voter can cast their vote only once, maintaining the integrity and fairness of the election process. Additionally,
the system allows administrators to monitor election activities and generate voting results efficiently.
Overall, this project aims to provide a secure, user-friendly, and reliable electronic voting system for Debre Tabor University’s student union elections.
User Roles
Administrator Functions
Login: Access admin features with valid credentials.
User Management: Create, activate, or deactivate user accounts.
View Results & Reports: See election results and generate reports on voter turnout, vote distribution, and logs.
Manage Comments: View/delete voter feedback.
Set Dates & Post Announcements: Set voting dates and share election-related news.
Election Committee Functions
Login: Access committee features with credentials.
Register Voters/Candidates: Add voters and candidates during registration periods.
Manage Election Schedule: Set election dates and candidate lists.
View Comments & Results: Access voter feedback and election outcomes.
Candidate Requests & Reports: Send requests to discipline committee and generate reports.
Discipline Committee Functions
Login: Access discipline features.
Verify Candidate Eligibility: Check disciplinary records.
Manage Candidate Records & Reports: View/edit records and generate reports.
Voter Functions
Login & Cast Vote: Secure voting (one vote per election).
Account Security: Change password.
View Results & Candidate Lists: See outcomes and candidate info.
Send Comments: Provide feedback.
Candidate Functions
Login & View Results: Access account and election outcomes.
Account Security: Change password.
View Candidate Lists & Post Announcements: Access info and share news.
Send Comments: Submit feedback.
Class Representative Functions
Login: Access account.
Nominate Class Reps: Departments submit potential candidates to the election committee.
System-Wide Functions
Automatic Vote Counting: Ensures accurate results.
Prevent Double Voting: One vote per student rule enforced.
Logout: Secure session termination for all users.
Technologies Used
Frontend: HTML, CSS, JavaScript
Backend: PHP
Database: MySQL
Server: XAMPP / Apache
